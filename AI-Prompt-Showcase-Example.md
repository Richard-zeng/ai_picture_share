# AI 图片提示词作品集示例

> 这是一个示例页面，演示如何在 GitHub 中展示本地图片和提示词案例。

## 这个文件是做什么的

如果你第一次写这类 Markdown，可以先参考这个示例，再把里面的内容替换成你自己的。

## 建议目录结构

```text
AIpicture/
├─ README.md
└─ images/
   ├─ case-01/
   │  ├─ image-01.png
   │  └─ image-02.png
   └─ case-02/
      └─ image-01.png
```

## 为什么图片能显示

因为 Markdown 里写的是相对路径，例如：

```md
![梦幻森林少女 1](./images/case-01/image-01.png)
```

这表示：
- 当前 Markdown 文件在仓库根目录
- 图片在 `images/case-01/` 这个文件夹里
- 只要你上传到 GitHub 时，文件结构没变，图片就能正常显示

## 案例 01：梦幻森林少女

### 生成图片

![梦幻森林少女 1](./images/case-01/image-01.png)
![梦幻森林少女 2](./images/case-01/image-02.png)

### 提示词

```text
一个站在薄雾森林里的少女，长发，白色裙子，阳光从树林缝隙洒下，电影感构图，细节丰富，梦幻氛围
```

### 使用模型

- GPT Image 1

### 简短说明

这一组主要想测试“梦幻感”和“森林光影”的表现。最后得到的图片整体比较柔和，适合做氛围感主题。

## 你可以怎么改成自己的

你只要替换下面这些内容就够了：

- 把标题换成你自己的案例标题
- 把 `./images/...` 路径换成你真实的图片路径
- 把提示词换成你自己的
- 把模型名换成你实际使用的模型
- 把说明换成你自己的观察或总结

## 最后提醒

如果 GitHub 上图片没有显示，优先检查这 3 件事：

1. 图片文件有没有真的放进仓库
2. 图片路径有没有写错
3. 文件名大小写和后缀名有没有对应上
